# AGENTS.md

## Project Shape

- This is a TanStack Start app with a Bun runtime; the app also exposes the proxy routes.
- UI routes and proxy/API routes both live under `src/routes/*` and run through Vite/TanStack Start on port `1455` with `bun run dev`.
- Browser OAuth uses the Codex client redirect URI `http://localhost:1455/auth/callback`; keep the unified TanStack Start server on this port instead of adding a separate callback server.
- Proxy route files use `createFileRoute(...){ server: { handlers } }` and delegate shared logic to `src/proxy/handlers.ts`.
- `src/routeTree.gen.ts` is generated by TanStack Router. Do not edit it manually.

## Commands

- Install/use Bun; `bun.lock` is the active lockfile.
- Unified UI/proxy dev server: `bun run dev`.
- Production build: `bun run build`.
- Typecheck: `bun run typecheck`.
- Lint: `bun run lint`.
- Format: `bun run format`.
- Tests exist as a script (`bun run test`) but there are currently no repo-specific tests to rely on.

## Verification

- For UI changes, run `bun run typecheck`, `bun run lint`, and `bun run build`.
- For proxy changes, also smoke-test the TanStack server: start `bun run dev`, then request `GET /health` and `GET /v1/models` on the same port.
- Build output lands in `.output/` and is ignored by ESLint via `eslint.config.js`; do not edit or commit generated build artifacts.

## Proxy Notes

- Enabled models are currently hard-coded in `src/proxy/model-registry.ts`.
- Routing is model-id driven: requests must include a string `model` field that matches an enabled registry entry.
- OpenRouter models only support `/v1/chat/completions` in the current adapter.
- OpenAI pool models support `/v1/responses` and `/backend-api/codex/responses`; connected ChatGPT account tokens are used before falling back to env API keys.
- Env vars used by the proxy: `OPENROUTER_API_KEY`, `CMM_OPENAI_KEYS`, and `OPENAI_API_KEY`.

## UI Notes

- Use shadcn/base components from `src/components/ui/*`; add components with `bunx --bun shadcn@latest add ...` rather than hand-copying registry code.
- Import project code through the `@/*` alias from `tsconfig.json`.
- Global theme tokens live in `src/styles.css` using Tailwind v4 and OKLCH CSS variables.
- The product direction is captured in `PRODUCT.md`: product-register, clean/minimal SaaS, avoid bubbly or overly-spaced dashboard UI.

## Formatting And Style

- Prettier uses no semicolons, double quotes, LF endings, and the Tailwind plugin from `.prettierrc`.
- TypeScript is strict and fails unused locals/params; remove unused imports instead of leaving placeholders.
